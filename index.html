No-No's

  1. Do not place non-.column elements as immediate children of .row. Evil things will happen if you choose to do so.

  2. Until support for IE9 is dropped, avoid using .row without .column. If the content does not intend to share horizontal space, a simple div should be used for “block” containment instead of .row.

  3. When declaring justify-{{ start | center | end }} on .row, it is important to also specify text-{{ alignment }} on .column for the FlexGrid fallback to work properly.

  4. Never try to apply box-model properties (i.e. padding, borders, or margin) directly to .column. Because this is a flexed child, these styles will render differently (and incorrectly) in different browsers. The solution is to apply these styles to .inner-column. This practice occurs automatically in FlexGrid by passing class properties applied to .column to its immediate child, .inner-column.

  5. Because .column has flexed-child properties, it must be directly nested inside of a flex-parent (i.e. .row) for it’s flexed properties to take effect. For this reason, do not use .column without .row.
